    这篇教程以图文方式，记录了从零开始，使用 Github Page 和 Hexo 搭建博客的过程。
    大部分资料来源https://www.cnblogs.com/Narcissus-why/p/9030820.html
    在此感谢作者.我在末尾加上自己的一些内容和节省了一些个人觉得可以省略的步骤,下面开始正文

    1.开通GitHub

    浏览器打开https://github.com 注册账号
    登陆 Github ，点击页面右上角的 “+” 号，再点击 New repository 创建一个新的仓库：

    在新的页面中，将 Repository name （仓库的名称）设置为 username.github.io ，其中 username 是你的用户名，这里我的用户名为 hyweng ，所以设置为 hyweng.github.io。

    点击页面下方的 “Create repository” ，创建这个仓库。

    仓库创建完成后，你的博客的静态页面就可以被托管在这个仓库中，可以通过连接 username.github.io 访问博客。

    当然，现在访问这个连接的话只会得到一个 404 页面，描述是 “There isn’t a GitHub Pages site here.” ，这是因为我们现在还没有向仓库里添加内容：

    2.下载 & 安装 & 配置 Git

    Hexo 是一个基于 Node.js 的静态站点生成器，但如果要将站点 publish 到 Github 上的话，我们还需要依赖 Git 。

    我们也可以通过 Git 来简单地测试我们的 Github pages 是否申请成功。

    2.1 下载 & 安装

    Git 可以在这个页面上下载（Windows）： https://git-scm.com/download/win （不建议使用 IE 访问）

    （通常而言，）在打开上面那个页面时，网页会自动地帮你选择版本与你的系统相符的 Git 进行下载。

    下载完毕后双击安装包进行安装：

    安装时一路 “Next” 即可。

    安装完成后点击 “Finish” 结束安装（取消 View Release Notes 这个选框，少打开一个浏览器窗口）：

    2.2 配置

    2.2.1 生成 SSH 密钥对

    SSH 密钥可以让我们在 publish 博客时，绕过 Github 的用户名和密码验证。
    在桌面上点击右键，点击 “Git Bash Here” 打开 bash ：

    输入cd ~进入用户主目录：

    执行 ssh-keygen -t rsa -C "user@server.com" 生成密钥对，其中-t rsa指定了密钥的类型为 RSA ，-C "user@server.com"为密钥添加一段注释，可以填写为你注册 Github 时所使用的邮箱。
    程序在生成密钥对时，会要求你输入一个文件路径：

    上图括号内的是默认文件路径，我建议在默认路径的基础上添加一个后缀作为最后的文件名，这里我添上了 “_hyweng.github.io” ，也就是我的 Github Pages 仓库的名字，你可以添上自己喜欢的后缀，比如说你的 Github Pages 仓库名。

    接下来还会让你输入密码，这里我们直接敲两下回车略过即可，我们设置密钥对的目的本来就是为了免验证：

    执行 ls -l .ssh 可以看见我们需要的密钥已经生成了：

    其中 “id_rsa_hyweng.github.io” 为私钥， “id_rsa_hyweng.github.io.pub” 为公钥。

    2.2.2 将公钥导入 Github

    执行 cat .ssh/id_rsa_hyweng.github.io.pub 打印公钥的内容（文件名可能有所不同），然后将输出复制进剪切板中（选取输出的全部内容，按 Ctrl + Insert ）：

    接下来，我们回到 Github ，点击右上角的头像，再点击 “Settings” 进入设置页面：

    进入设置页面后，点击 “Personal settings” 下的 “SSH and GPG keys” ，再点击右侧的 “New SSH key”：

    接下来我们输入 Title 和 Key 的内容，Title 可以按照你自己的喜好来填，比方说 “github page blog” 或者 “my first key” ，这里我填写为 Github Pages 仓库的名称。

    输入完 Title 之后，我们将刚才复制进剪切板的公钥粘贴在 Key 中，然后点击下面的 “Add SSH key” 按钮。

    接下来 Github 可能会要求你输入密码以继续操作，这个密码就是你的 Github 账户的密码。

    完成后，你会回到刚才的 “SSH and GPG keys” 页面，如果成功的话会看见页面上多出了一个新的 SSH key：

    至此，公钥导入成功。接下来我们回到 Git Bash ，我们还需要对客户端做一些配置。

    2.2.3 SSH 客户端配置

    首先，我们要将新产生的私钥添加进 ssh-agent 的高速缓存中。

    还是在刚才的 bash 中，执行 ssh-agent bash 使用 ssh-agent 启动 bash，接着执行 ssh-add .ssh/id_rsa_hyweng.github.io

    接着，由于我们在创建密钥对时没有使用默认的文件名，因此需要配置一下 Host 和 密钥间的对应关系。
    执行vi .ssh/config，在文件内添加如下：

    Host hyweng.github.io
    HostName github.com
    IdentityFile ~/.ssh/id_rsa_hyweng.github.io

    这里 Host 我设置成了我的 Github Pages 的地址，你也可以改成你自己喜欢的其他值。

    HostName 必须是 github.com 。

    IdentityFile 指向我们刚才生成的密钥对中的私钥文件。

    保存文件并退出，此时我们的 SSH 客户端就知道，当我们连接 hyweng.github.io 时，实际使用 id_rsa_hyweng.github.io 这个私钥连接到 github.com 这个主机。

    接下来我们可以测试一下连接。
    执行 ssh -T git@hyweng.github.io，当出现 “(yes/no)” 时输入 “yes” 同意，正常情况下会出现 “…You’ve successfully authenticated…” ，表示连接可以正常建立：

    最后，在配置完密钥对之后，我们还需要设置一下 Git 的用户名和邮箱。

    执行 git config --global user.name "username"，这里我们把 “username” 填写为 Github 账户名，你也可以填写成其他名字。

    执行 git config --global user.email "user@server.com"，这里我们把 “user@server.com“ 填写为注册 Github 时使用的邮箱，你也可以填写为其他邮箱。

    以上两个命令在参数和执行正确的情况下都不会产生输出。

    3.安装 Node.js 和 Hexo

    3.1 安装 Node.js

    Hexo 基于 Node.js ，所以为了我们漂亮的博客，我们还需要装一个 Node.js 。

    我们可以在这个页面上下载 Node.js 的安装包： http://nodejs.cn/download/

    点击上述连接，由于我的是64位的 win10 系统，所以我们选择 node-v12.16.1-x64.msi 这一项：

    下载完毕后，双击打开安装包进行安装：

    接下一路 Next 和 Accept terms in the License Agreement ，安装完成后点击 Finish 结束安装。

    然后打开命令提示符，依次执行 node -v 和 npm -v ，正常情况下这两条命令会输出 Node.js 和 npm 包管理器的版本号。

    4.建站

    4.1 站点初始化

    现在我们要在硬盘上找一个地方来安放我们的博客，假设我们打算把博客放在 C 盘根目录下的 “hexo_blog” 文件夹里（也可以放在其他你喜欢的地方）。

    在 cmd 中依次执行 C: 和 cd C:/，切换到 C 盘根目录，然后执行 hexo init hexo_blog （如果已经存在一个叫 “hexo_blog” 的文件夹的话，把旧的文件夹删除）：

    初始化完成后，依次执行 cd hexo_blog 和 npm install 安装依赖：

    4.2 本地浏览

    有时，我们希望在将博客 publish 到 Github 之前，首先在本地看一下效果如何。为了达成这个目的，我们需要启动一个本地服务器。

    还是在我们站点的根目录（ hexo_blog ）下，依次执行：

    hexo clean # 清理缓存
    hexo generate # 生成静态文件
    hexo server –port 3000 # 将服务器运行在端口 3000 上

    4.3 部署至 Github

    4.3.1 #Site

    使用你喜欢的文本编辑器打开 “_config.yml” 文件，在 #Site 下可以修改站点信息（下面是我的配置）：

    # Site
    title: 瓜果仁
    subtitle:
    description: 一粒代码的博客
    keywords:
    author: codetroupes
    language: zh-CN
    timezone: Asia/Shanghai

    关于 #Site 部分各字段的描述请参考官方文档：https://hexo.io/zh-cn/docs/configuration.html
    需要注意两点：
    一、如果某项配置不为空，则该项配置的冒号后面必须跟上一个半角空格（英文空格）。
    二、 keywords 中，两个相邻的关键字使用英文逗号分隔。

    4.3.2 #URL

    在 #URL 下，修改 url 为你的 Github Pages 地址：

    # URL
    ## If your site is put in a subdirectory, set url as ‘http://yoursite.com/child' and root as ‘/child/‘
    url: https://codetroupes.github.io
    root: /
    permalink: :year/:month/:day/:title/
    permalink_defaults:
    pretty_urls:
    trailing_index: true # Set to false to remove trailing ‘index.html’ from permalinks
    trailing_html: true # Set to false to remove trailing ‘.html’ from permalinks

    4.3.3 #Deployment

    找到 #Deployment ，将其下的 deploy 项修改为：

    # Deployment
    ## Docs: https://hexo.io/docs/deployment.html
    deploy:
    type: git
    repo: git@codetroupes.github.io:codetroupes/codetroupes.github.io
    branch: master

    其中 type 填写为 git ，repo 填写为你的 Github Pages 所在仓库的地址， branch 填写为 master （仓库的一个分支，目前只有一个 master ）。
    需要注意一下 repo 的填写。为了用上我们早前设置的密钥，我们需要把 repo 的连接写成 SSH 协议的格式：

    git@host:username/repo_name.github.io

    其中 host 为我们在 .ssh/config 文件中设置的 Host ， username 为你的 Github 用户名， repo_name 为你的仓库名称（ Github Pages 仓库）。
    修改完毕后保存并关闭文件

    4.3.4 安装 hexo-deployer-git & 部署至 Github

    最后，我们需要一个叫 hexo-deployer-git 的 npm 包帮我们将网站部署到 Github 上。
    现在我们的 cmd 应该还在本地运行我们的博客网站，按 Ctrl + C 终止运行 ：

    我们的 cmd 应该还在博客文件夹的根目录下，执行 npm install hexo-deployer-git --save 安装 hexo-deployer-git：

    安装完成后，依次执行下列命令进行部署：

    hexo clean
    hexo generate
    hexo deploy

    如果 SSH Key 配置正确的话，这里不会要求你输入用户名和密码。部署完成后：

    这时我们打开自己的 Github Pages ，就可以看见新鲜出炉的博客了~ https://codetroupes.github.io

    5.文章管理

    文章的 Markdown 文件存放在博客文件夹下的 source 目录下。
    默认情况下，新建的文章都会存放在 “_post” 子目录下，现在里面只有一个叫 “hello-world.md” 的文件，这就是我们刚刚 publish 到 Github 上的博客中的唯一一篇文章的 Markdown 源文件。

    5.1 新建/删除文章

    回到 cmd ，在博客文件夹的根目录下执行 hexo new <title> 来向 post 中添加一篇指定标题的空白文章，例如 hexo new “my first article” ：

    在 source/_post 下会生成一个与之对应的新 Markdown 文件，这就是我们的新文章，选择你喜欢的文本编辑器对它进行编辑即可~

    编辑并保存之后，在 cmd 中依次执行

    hexo clean
    hexo generate
    hexo deploy

    即可将新文章部署至 Github 。

    注：从博客被 publish 到页面的更新可见，中间有一段延迟。所以如果发现 deploy 之后页面还是和原来一样，不要慌张，请稍等一两分钟。（ Github 有自己的 CDN ）

    删除文章也很简单，只需要在执行 hexo clean前删掉对应的 Markdown 文件即可。
